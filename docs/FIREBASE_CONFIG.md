# Firebase Configuration Strategy

This document clarifies the Firebase configuration patterns used across the Vehicle Vitals monorepo.

## Configuration Files Overview

### 1. Web Application: `packages/web/src/shared/firebaseConfig.js`
- **Purpose**: Web-specific Firebase client configuration
- **Environment**: Vite (uses `import.meta.env`)
- **Features**: Dynamic initialization to prevent build issues
- **Used by**: React web application only

### 2. Shared Package: `packages/shared/src/firebaseConfig.js`
- **Purpose**: Cross-platform Firebase client configuration
- **Environment**: Node.js/Expo-compatible (uses `process.env`)
- **Features**: Platform detection for Expo vs Node.js
- **Used by**: Mobile app and Node.js environments

### 3. API Server: `packages/api-server/src/services/firebase.ts`
- **Purpose**: Server-side Firebase Admin SDK configuration
- **Environment**: Node.js server environment
- **Features**: Admin SDK initialization with service account
- **Used by**: API server for server-side operations

## Usage Guidelines

### When to use which config:

- **Web app components**: Use `packages/web/src/shared/firebaseConfig.js`
- **Mobile app**: Use `packages/shared/src/firebaseConfig.js`
- **API server**: Use `packages/api-server/src/services/firebase.ts`
- **Shared utilities**: Use appropriate config based on context

### Environment Variables:

**Web (Vite):**
```bash
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_project_id
# ... etc
```

**API Server:**
```bash
FIREBASE_PROJECT_ID=your_project_id
FIREBASE_CLIENT_EMAIL=firebase-adminsdk-xxxxx@your_project.iam.gserviceaccount.com
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"
```

**Mobile (Flutter):**
Uses `mobile/lib/firebase_options.dart` generated by FlutterFire CLI.

## Migration Notes

- Web package no longer imports from shared Firebase config
- Shared package is primarily for mobile/cross-platform usage
- API server uses Firebase Admin SDK independently
- No more circular dependencies between configs
